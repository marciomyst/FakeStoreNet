# 07. Add PostgreSQL Provider Support

## Description
Implement support for the PostgreSQL database in the infrastructure layer, allowing flexible configuration via connection strings.  
Rationale: provide a robust and widely adopted production alternative, ensuring performance and scalability.

## Acceptance Criteria
- PostgreSQL provider configuration is available in `appsettings.json` and environment variables.  
- `DbContext` uses the correct provider based on configuration (InMemory, SQLite, or PostgreSQL).  
- EF Core–generated SQL scripts execute without errors in PostgreSQL.  
- Integration tests validate basic CRUD operations against a PostgreSQL instance (e.g., using a Docker container).

## Technical Refinement

- Provider Configuration:
  - Read `Provider` (e.g., InMemory, SQLite, PostgreSQL) and connection string from `appsettings.json` and environment variables.
  - Configure `DbContextOptionsBuilder` in `Startup`/`Program`:
    ```csharp
    if (provider == "PostgreSQL")
      options.UseNpgsql(connString, npgsql => npgsql.MigrationsAssembly(typeof(YourContext).Assembly.FullName));
    else if (provider == "SQLite")
      options.UseSqlite(connString);
    else
      options.UseInMemoryDatabase("TestDb");
    ```
  - Validate connection string presence and fail fast on startup errors.

- EF Core Migrations:
  - Use `dotnet ef migrations add InitialPostgres` and `dotnet ef database update` for PostgreSQL.
  - Ensure SQL scripts generated by `ef migrations script` are compatible with PG syntax (check serial vs identity).
  - Use `Npgsql.EntityFrameworkCore.PostgreSQL` package with correct version compatibility.

- Docker Compose Integration:
  - Configure `docker-compose.yml` to include a `db` service for Postgres with persistent volume and healthcheck.
  - Use network aliases to allow the application to connect using service name.
  - Provide example `docker-compose.override.yml` for local development with credentials and ports.

- Integration Testing:
  - Use `DotNet.Testcontainers` or a Docker-managed PostgreSQL container in tests.
  - Run migrations at test startup to prepare schema.
  - Verify CRUD operations against container instance.
  - Clean up database between test runs for isolation.

- Performance & Pooling:
  - Enable Npgsql connection pooling via connection string parameters (e.g., `MaxPoolSize`).
  - Configure `CommandTimeout` and retry policies using `EnableRetryOnFailure`.

- Documentation:
  - Update `README.md` with setup instructions for Postgres support and migration commands.
